# lbh939.com 홈페이지 개편 작업 지시서

## 0. 이 작업의 목적 (꼭 먼저 읽기)

지금 `lbh939.com` 메인은 첫 문구와 카드 구성이 "투자 사이트"처럼 보인다.
이걸 **"1인 개발자 lbh939의 앱 포트폴리오"** 느낌으로 무게중심을 옮기고,
실제로 만든 프로젝트(텔레그램 봇 · 카카오톡 봇 · 러닝뷰)를 정확하게 추가하는 작업이다.

### 절대 규칙 (위반 시 작업 실패로 간주)

1. **사실만 쓴다.** 이 문서 8번 "검증된 사실" 항목에 적힌 내용만 사용한다.
   기능을 추측하거나, 없는 기능을 있는 것처럼 미화하거나, 출시 안 한 걸 출시한 것처럼 쓰지 않는다.
2. **하드코딩·눈속임 금지.** 디자인 토큰(색·여백·폰트)은 반드시 기존 `app/globals.css`의 CSS 변수를 쓴다.
   `#1B64DA` 같은 색을 직접 박지 말고 `var(--accent)` 등 기존 변수를 그대로 쓴다.
3. **기존 디자인 시스템을 그대로 따른다.** 새 상세 페이지는 기존 `app/quantapp/page.tsx` 구조와
   `components/Header.tsx`, `components/Footer.tsx`, `components/LegalLayout` 패턴을 본떠서 만든다.
   새로운 색·새로운 컴포넌트 스타일을 임의로 만들지 않는다.
4. **다크/라이트 모드 둘 다 정상 동작해야 한다.** (기존 사이트가 둘 다 지원함)
5. **확인이 필요한 부분은 코드에 TODO 주석으로 남기고, 작업 끝에 사람에게 질문으로 보고한다.** (예: 정확한 봇 아이디)

## 4. 메인 페이지 카드 4장 (정확한 내용)

각 카드는 기존 퀀트뷰 카드의 마크업 패턴(아이콘 박스 + 제목 + 상태 배지 + 한 줄 설명 + "자세히 보기")을 그대로 따른다.
아이콘은 기존에 쓰는 `lucide-react`에서 의미에 맞는 것을 고른다 (예시는 참고용, 더 맞는 게 있으면 사용 가능).

| 순서 | 제목 | 상태 배지 | 한 줄 설명 | 링크 | 아이콘(참고) |
|------|------|-----------|------------|------|--------------|
| 1 | 퀀트뷰 | 출시 | 미국 주식 AI 분석 도구 | `/quantapp` | 기존 "Q" 유지 |
| 2 | 텔레그램 봇 | 운영 중 | 퀀트뷰의 분석을 텔레그램에서 받는 봇 | `/telegram-bot` | `Send` |
| 3 | 카카오톡 봇 | 개발 중 | 단톡방에서 시세·환율·코인·유튜브 요약을 답해주는 봇 | `/kakao-bot` | `MessageCircle` |
| 4 | 러닝뷰 | 제작 중 | 러닝·마라톤 일정과 기록을 담을 앱 (준비 중) | `/running-view` | `Footprints` 또는 유사 |

상태 배지 색 가이드 (기존 globals.css 변수 활용):
- `출시`, `운영 중` = 기존 퀀트뷰 배지와 동일한 강조색 계열(`var(--accent)`)
- `개발 중`, `제작 중` = 강조색이 아닌 차분한 보조색 계열(`var(--secondary)`/`var(--section-bg)` 등)으로 구분
  → 출시된 것과 준비 중인 것이 한눈에 구분되게.

같은 기존 구조를 본떠서 만든다. **단, 아직 실제 동작 기능이 없는 초기 단계임을 정직하게 표현한다.**

- 다운로드/시작 버튼 없음 (Play Store 미출시, 도메인 미발급).

### 만들 계획 (기능이 아니라 "계획"으로 명확히 표기)
섹션 제목을 "핵심 기능"이 아니라 **"준비 중인 기능"** 또는 **"이런 앱을 만들고 있어요"** 로 한다.
- 마라톤 일정: 여러 마라톤 대회 일정을 한곳에서 조회 (계획)
- 러닝 다이어리: 러닝 기록을 남기고 관리 (계획)
- AI 코치: 기록을 바탕으로 코칭 제공 (계획)
- 러닝 커뮤니티: 러너들과 소통 (계획)

### 상태 안내
- 안내 문구(솔직하게): `러닝뷰는 현재 초기 개발 단계입니다. 출시되면 이곳에서 안내드리겠습니다.`

## 8. 검증된 사실 (이 내용만 사용 — 추측 금지)

> 아래는 실제 소스코드를 직접 확인해 정리한 사실이다. 페이지 문구는 이 범위 안에서만 작성한다.

### 텔레그램 봇 (E:\quant_app)
- 퀀트뷰와 **같은 git 저장소 · 같은 SQLite DB(`data/app.db`) · 같은 `core/` 분석 모듈 · 같은 Oracle 서버**를 공유한다. 즉 퀀트뷰의 별도 앱이 아니라 "같은 서비스의 다른 입구"다.
- Production에서 상시 운영 중(`quantapp-bot.service`).
- 실제 구현 명령어: 종합 분석(/ana), 오늘의 신호(/sig), 물타기(/avg), 통계 승률(/prob), 뉴스(/news), 매크로(/macro), 관심종목 추가·삭제·목록(/add·/del·/list), 가입(/join 초대코드), 관리자 명령(/invite·/kick·/userlist).
- **초대코드 기반 비공개 봇.** 누구나 자유 가입 불가.
- 공개 아이디 후보 2개(`@byeongha_quant_bot` / `@QuantBot_bot`) — 확정 안 됨 → 페이지에 아이디 노출하지 않음.

### 카카오톡 봇 (E:\kakao_bot)
- 운영자 본인 카카오톡 단톡방에서만 동작하는 봇. **공개 채널·봇 아이디 없음.**
- 개발 중(아키텍처 전환 단계, 공개 출시 전).
- 실제 구현 기능: 환율, 공포·탐욕 지수, 코인 시세, 봇 상태 점검, 국내 주식 시세(종목코드/한글명), 해외 주식 시세(티커), 유튜브 링크 자동 요약.
- 기술: Python + FastAPI + 메신저봇R(에뮬레이터 연동). (페이지에는 기술 스택을 굳이 노출하지 않아도 됨)
- 단톡방 이름 등 비공개 정보는 노출 금지.

### 러닝뷰 (E:\running_view)
- **실제 동작 기능 0개.** 현재 `flutter create` 기본 데모 코드(`lib/main.dart`)만 존재. 백엔드 폴더 없음.
- Play Store 미출시, 도메인 미발급(계획 도메인 `runningview.lbh939.com`은 아직 미배포).
- 계획: 마라톤 일정 조회 + 러닝 다이어리 + AI 코치 + 러닝 커뮤니티 (안드로이드 우선).
- 따라서 "제작 중 / 준비 중"으로만 표기하고, 기능은 전부 "계획"으로 명시한다.

## 9. 함께 점검할 자잘한 것

- 새 페이지를 추가하면 `sitemap.xml`(자동 생성 설정)에 자연히 포함되는지 확인. 수동 등록이 필요한 구조면 새 경로 3개를 추가한다.
- 각 새 페이지에 기존 페이지처럼 `metadata`(title/description)를 적절히 넣는다.
- 메인 카드에서 새 상세 페이지로 가는 링크, 상세 페이지의 "메인으로" 링크가 정상 동작하는지 확인.
- 모바일·데스크톱 둘 다에서 카드 4장이 깨지지 않게 그리드 정렬 확인(기존 반응형 패턴 활용).

## 10. 마지막 검수 (작업을 다 한 뒤 반드시 수행)

1. `npm run build` 가 에러 없이 통과하는지 확인한다. 에러가 나면 근본 원인을 고친다. (눈속임·임시방편 금지)
2. `npm run lint` 를 돌려 경고/에러를 정리한다.
3. 추가·수정한 모든 페이지를 다시 열어, 8번 "검증된 사실"과 어긋나는 문구가 없는지 한 줄씩 대조한다.
   특히 ① 없는 기능을 있는 것처럼 쓴 곳 ② 출시 안 한 걸 출시했다고 쓴 곳 ③ 비공개 봇에 공개 링크를 단 곳 이 없는지 집중 확인한다.
4. 다크 모드·라이트 모드를 모두 전환해보고 글자/배경 대비가 깨지지 않는지 확인한다.
5. 위 1~4를 마친 뒤, **무엇을 바꿨는지 요약**과 **사람 확인이 필요한 항목(예: 텔레그램 봇 공개 아이디 노출 여부)**을 정리해 보고한다.

> 코딩을 다 한 뒤에는 위 검수 절차를 빠짐없이 수행하고, 잘 코딩했는지 스스로 다시 확인한 다음 최종 결과를 보고할 것.
