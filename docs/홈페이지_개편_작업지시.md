# lbh939.com 홈페이지 개편 작업 지시서

## 0. 이 작업의 목적 (꼭 먼저 읽기)

지금 `lbh939.com` 메인은 첫 문구와 카드 구성이 "투자 사이트"처럼 보인다.
이걸 **"1인 개발자 lbh939의 앱 포트폴리오"** 느낌으로 무게중심을 옮기고,
실제로 만든 프로젝트(텔레그램 봇 · 카카오톡 봇 · 러닝뷰)를 정확하게 추가하는 작업이다.

### 절대 규칙 (위반 시 작업 실패로 간주)

1. **사실만 쓴다.** 이 문서 8번 "검증된 사실" 항목에 적힌 내용만 사용한다.
   기능을 추측하거나, 없는 기능을 있는 것처럼 미화하거나, 출시 안 한 걸 출시한 것처럼 쓰지 않는다.
2. **하드코딩·눈속임 금지.** 디자인 토큰(색·여백·폰트)은 반드시 기존 `app/globals.css`의 CSS 변수를 쓴다.
   `#1B64DA` 같은 색을 직접 박지 말고 `var(--accent)` 등 기존 변수를 그대로 쓴다.
3. **기존 디자인 시스템을 그대로 따른다.** 새 상세 페이지는 기존 `app/quantapp/page.tsx` 구조와
   `components/Header.tsx`, `components/Footer.tsx`, `components/LegalLayout` 패턴을 본떠서 만든다.
   새로운 색·새로운 컴포넌트 스타일을 임의로 만들지 않는다.
4. **다크/라이트 모드 둘 다 정상 동작해야 한다.** (기존 사이트가 둘 다 지원함)
5. **확인이 필요한 부분은 코드에 TODO 주석으로 남기고, 작업 끝에 사람에게 질문으로 보고한다.** (예: 정확한 봇 아이디)
6. **한국어 톤은 표준어 + 존댓말로 통일한다.** 신규 페이지·신규 섹션 모든 한국어 문구는 존댓말로 작성하고, "박다 / 박혀있다 / 흐름 / 정합 / 자체 / 짚다 / 깔다" 같은 비표준 표현을 쓰지 않는다.

| 순서 | 제목 | 상태 배지 | 한 줄 설명 | 링크 | 아이콘(참고) |

| 1 | 퀀트뷰 | 출시 | 미국 주식 AI 분석 도구 | `/quantapp` | 기존 "Q" 유지 |
| 2 | 텔레그램 봇 | 운영 중 | 퀀트뷰의 분석을 텔레그램에서 받는 봇 | `/telegram-bot` | `Send` |

> 3차 정정 (2026-05-25): 카카오톡 봇 = "개발 중" → "운영 중" 으로 변경. 본인 운영 단톡방 안에서 상시 동작 중이라는 점을 반영. 메인 페이지 카드 배지·상세 페이지 Hero 배지·아이콘 박스 색 모두 텔레그램 봇과 동일하게 강조색 계열로 통일.

- `출시`, `운영 중` = 기존 퀀트뷰 배지와 동일한 강조색 계열(`var(--accent)`)

  → 출시·운영 중인 것과 준비 중인 것이 한눈에 구분되게.

## 4-A. About 섹션 (= 메인 페이지 신규 추가)

### 위치
- Apps 섹션과 Contact 섹션 **사이**에 둔다 (= 자연스러운 흐름).
- 기존 다른 섹션과 같은 좌우 여백·max-w·패딩 구조를 그대로 따라가서 시각 흐름을 유지한다.
- 배경은 흰/검정(`var(--background)`) 그대로 두거나 `var(--section-bg)` 둘 중 하나를 골라 Apps·Contact 와 번갈아 보이게 한다.

### 내용
- 섹션 제목: `About` (또는 `만든 사람` 같은 한글 표현 — 자연스러운 쪽으로 선택, 단 짧게).
- **핵심 메시지(이 문장 그대로, 존댓말)**: `제가 실제로 쓰고 싶은 도구를 직접 만듭니다!`
  → 큰 문구로 시각적 무게를 준다(예: 본문보다 큰 굵은 글씨).
- 핵심 메시지 아래 보조 설명 (2~4문장 / 담백하면서 약간 재미있고 친근한 토스 스타일 / 과한 미사여구 금지):
  - 1인 개발자가 기획·디자인·개발·배포까지 혼자 한다는 점을 자연스럽게 담는다.
  - 투자·일상·러닝 같이 본인이 실제로 쓰는 도구를 만든다는 톤.
  - "잘 만들기보다 진짜로 쓰게 만든다" 같은 느낌(=원문 그대로 박지는 말고 자연스럽게 변주 가능).
  - 초안 예시(=수정 가능):
    > 기획부터 디자인, 개발, 배포까지 혼자 합니다. 잘 팔리는 앱보다 제가 매일 쓰고 싶은 앱을 먼저 만듭니다. 투자, 일상, 러닝까지 — 필요한 도구가 없으면 직접 만들어 보는 1인 개발자입니다.
  - 위 초안 문구는 나중에 사람이 쉽게 손볼 수 있도록 컴포넌트 안 별도 상수(예: `ABOUT_BODY`)에 두지 않아도 되지만, 본문 영역만 잘 식별되게 작성한다.

### GitHub 주소 처리 (= 매우 중요)
- 깃허브 주소(`https://github.com/lbh939-ai`)는 **코드에는 넣되 화면에는 노출하지 않는다**.
  - 현재 저장소가 비공개라 외부에서 클릭 시 404가 떠 보이기 때문이다.
- 구체 방식:
  1. JSX 안에 깃허브 카드/버튼 코드를 작성하되 **JSX 주석(`{/* ... */}`)으로 둘러싸 화면에 안 보이게** 처리한다.
  2. 그 위에 `{/* TODO: 저장소를 공개로 전환하면 본 주석 한 줄만 해제해 GitHub 카드를 노출 */}` 같은 TODO 주석을 함께 둔다.
  3. 깃허브 카드 톤은 기존 Contact 섹션 카드(아이콘 + 라벨)와 동일 스타일로 미리 만들어 두면 나중에 주석만 풀어 바로 노출 가능.
- 깃허브 주소 자체 노출 외 다른 외부 링크는 About 섹션에 추가하지 않는다.

### 톤
- About 섹션도 본 차수 톤 통일 규칙(= **존댓말 + 담백하면서 친근한 토스 스타일**)을 따른다.
- 과한 이모지나 미사여구는 금지. 가벼운 톤 1~2 정도만 허용(= 핵심 메시지 끝 느낌표 같은 정도).

### 왜 만들었는지 / 어떻게 쓰는지 (= 신규 보강 섹션)
사실 범위 안에서만 짧게 안내한다. 본문 1~2단락 정도.
- 왜 만들었는지: 퀀트뷰 앱을 켜지 않아도 텔레그램에서 같은 분석을 빠르게 받고 싶어서 만들었다는 톤.
- 어떻게 쓰는지: 봇에 명령어 한 줄을 보내면 분석 결과를 채팅으로 받아볼 수 있다는 흐름 (예: `/ana TSLA` 한 줄을 보내면 종합 분석 결과를 채팅으로 응답). 단 명령어 그대로 노출보다 "한 줄 명령어로 분석 결과를 받아본다" 같은 자연스러운 설명 위주.

### 핵심 기능 (실제 구현된 명령어 기준 — 8번 사실 참고)
카드형으로 5~6개 정도만 골라 깔끔하게. 명령어 자체보다 "무엇을 해주는지"를 쉬운 말로.
- 종합 분석: 종목을 입력하시면 11단계 종합 분석 결과를 보내드립니다.
- 오늘의 신호: 관심·시스템 종목의 매수/매도 신호를 스캔해 알려드립니다.
- 물타기 분석: 보유 단가를 넣으시면 추가 매수 전략을 분석해드립니다.
- 통계 승률: 종목별 지표 기반 상승 확률 표를 제공해드립니다.
- 최신 뉴스: 종목 관련 최근 뉴스를 요약해드립니다.
- 매크로 요약: 주요 거시 지표(FRED 기반)를 정리해드립니다.

### 기술 스택 (= 신규 보강 섹션)
- 제목: `만든 기술` 또는 `사용 기술` (자연스러운 쪽으로 선택).

## 7. 상세 페이지 ③ 러닝뷰 (`app/running-view/page.tsx`)

같은 기존 구조를 본떠서 만든다. **단, 아직 실제 동작 기능이 없는 초기 단계임을 정직하게 표현한다.**
전체 문구 톤은 **존댓말 + 담백하면서 친근한 토스 스타일**로 통일한다.

### Hero
- 제목: `러닝뷰`
- 한 줄 소개: `러닝과 마라톤을 한 앱에서`
- 본문 설명(사실 기반 — 전부 "계획"임을 분명히):
  > 러닝뷰는 마라톤 일정 조회, 러닝 기록 다이어리, AI 코치, 러닝 커뮤니티를 한 앱에 담는 것을 목표로 준비 중인 안드로이드 앱입니다.
- 상태 표기: `제작 중`
- 다운로드/시작 버튼 없음 (Play Store 미출시, 도메인 미발급).

### 왜 만드는지 / 어떻게 쓰게 될지 (= 신규 보강 섹션)

- 어떻게 쓰게 될지: 출시 시점에는 안드로이드 앱으로 사용 (= 계획). 현재는 초기 개발 단계라 화면이나 사용 방법을 보여줄 수 없다는 점을 정직하게 명시.

### 만들 계획 (기능이 아니라 "계획"으로 명확히 표기)
섹션 제목을 "핵심 기능"이 아니라 **"준비 중인 기능"** 또는 **"이런 앱을 만들고 있어요"** 로 한다.
- 마라톤 일정: 여러 마라톤 대회 일정을 한곳에서 조회 (계획)
- 러닝 다이어리: 러닝 기록을 남기고 관리 (계획)
- AI 코치: 기록을 바탕으로 코칭 제공 (계획)
- 러닝 커뮤니티: 러너들과 소통 (계획)

### 기술 스택 (= 신규 보강 섹션 / 정직 표기 의무)
- 칩(태그) 형태로 한 줄 나열. 칩 디자인은 반드시 기존 `globals.css` 변수만 사용.

## 8. 검증된 사실 (이 내용만 사용 — 추측 금지)

> 아래는 실제 소스코드를 직접 확인해 정리한 사실이다. 페이지 문구는 이 범위 안에서만 작성한다.

### 텔레그램 봇 (E:\quant_app)
- 퀀트뷰와 **같은 git 저장소 · 같은 SQLite DB(`data/app.db`) · 같은 `core/` 분석 모듈 · 같은 Oracle 서버**를 공유한다. 즉 퀀트뷰의 별도 앱이 아니라 "같은 서비스의 다른 입구"다.
- Production에서 상시 운영 중(`quantapp-bot.service`).
- 실제 구현 명령어: 종합 분석(/ana), 오늘의 신호(/sig), 물타기(/avg), 통계 승률(/prob), 뉴스(/news), 매크로(/macro), 관심종목 추가·삭제·목록(/add·/del·/list), 가입(/join 초대코드), 관리자 명령(/invite·/kick·/userlist).
- **초대코드 기반 비공개 봇.** 누구나 자유 가입 불가.
- 공개 아이디 후보 2개(`@byeongha_quant_bot` / `@QuantBot_bot`) — 확정 안 됨 → 페이지에 아이디 노출하지 않음.

### 카카오톡 봇 (E:\kakao_bot)
- 운영자가 초대한 카카오톡 단톡방 안에서만 동작하는 비공개 봇. **공개 채널·봇 친구 추가 링크 없음.**
- **운영 중** (= 3차 정정 / 2026-05-25). 본인 운영 단톡방 안에서 상시 동작.
- 실제 구현 기능: 환율, 공포·탐욕 지수, 코인 시세, 봇 상태 점검, 국내 주식 시세(종목코드/한글명), 해외 주식 시세(티커), 유튜브 링크 자동 요약.
- 기술: Python + FastAPI + 메신저봇R(에뮬레이터 연동). (페이지에는 기술 스택을 굳이 노출하지 않아도 됨)
- 단톡방 이름 등 비공개 정보는 노출 금지.
- 이용 문의 안내 = 텔레그램 봇과 동일 형식 (= "초대받은 분만 이용 가능 / 카카오톡 채널로 문의" 안내 + 카카오톡 채널 메인 / 이메일 보조 두 버튼).

### 러닝뷰 (E:\running_view)
- **실제 동작 기능 0개.** 현재 `flutter create` 기본 데모 코드(`lib/main.dart`)만 존재. 백엔드 폴더 없음.

- 계획: 마라톤 일정 조회 + 러닝 다이어리 + AI 코치 + 러닝 커뮤니티 (안드로이드 우선).
- 따라서 "제작 중 / 준비 중"으로만 표기하고, 기능은 전부 "계획"으로 명시한다.

## 10. 마지막 검수 (작업을 다 한 뒤 반드시 수행)

1. `npm run build` 가 에러 없이 통과하는지 확인한다. 에러가 나면 근본 원인을 고친다. (눈속임·임시방편 금지)
2. `npm run lint` 를 돌려 경고/에러를 정리한다.
3. 추가·수정한 모든 페이지를 다시 열어, 8번 "검증된 사실"과 어긋나는 문구가 없는지 한 줄씩 대조한다.
   특히 ① 없는 기능을 있는 것처럼 쓴 곳 ② 출시 안 한 걸 출시했다고 쓴 곳 ③ 비공개 봇에 공개 링크를 단 곳 ④ 러닝뷰 기술 칩 안 "사용 예정"을 "사용 중"으로 잘못 표기한 곳이 없는지 집중 확인한다.
4. 다크 모드·라이트 모드를 모두 전환해보고 글자/배경 대비가 깨지지 않는지 확인한다.
5. 한국어 톤이 표준어 + 존댓말로 통일되어 있는지 검사한다. (= 모든 신규 페이지·신규 섹션 본문이 "~합니다 / ~해드립니다 / ~입니다" 형태로 통일 / "박다·박힘·흐름·정합·자체·짚다·깔다" 같은 비표준 표현 0건)
6. 위 1~5를 마친 뒤, **무엇을 바꿨는지 요약**과 **사람 확인이 필요한 항목(예: 텔레그램 봇 공개 아이디 노출 여부 / About 보조 설명 초안 수정 여부)**을 정리해 보고한다.

> 코딩을 다 한 뒤에는 위 검수 절차를 빠짐없이 수행하고, 잘 코딩했는지 스스로 다시 확인한 다음 최종 결과를 보고할 것.
